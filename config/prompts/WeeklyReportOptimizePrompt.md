# 团队周报优化任务（主管视角 - 通用版）
请帮我优化附件的团队周报JSON文件,作者视角要从“团队成员汇总”转换为“团队主管”。
## 一、核心原则
1. **保持原有JSON结构完全不变**：只替换优化后的内容，不改变任何字段层级。
2. **角色转换**：从“团队成员分别做了什么工作”转为“团队做了xxx工作”或者“带领或组织团队完成XXX工作”
3. **项目表述规范**：
   - 特定项目的工作用团队名义推进，不加“我指导/统筹”等修饰
   - 团队管理工作用本人名义（如“组织团队培训”、“建立服务群”）
   - 明确是本人的工作，也请忽略主语，直接使用”完成”作为开头（如“完成文档编写”、“参加会议”）
4. **精简要求**：去除重复项、合并同类项，保留核心信息，总长度控制在页面可以一次性完成输出。
5. **可读性**：按客户或项目类型进行分组，用空行分隔，层次清晰。

**必须从以下六个维度对本周工作进行深度总结，每个维度用【】标注主题，控制在2-3行：**
**提炼来源**：第一部分内容 + 第二部分中的里程碑、风险、能力建设相关内容
【整体进展】汇总本周整体报收/交付情况、重点项目群推进概况、成本/风险管控总体情况
【关键里程碑】提炼本周完成的重要里程碑（如验收、部署、功能开发完成、合同签约等）
【风险管控】汇总项目风险、异常情况、待解决问题及跟进状态
【能力建设】团队培训、技术沉淀、知识产出、流程优化等

【会议参与】重要会议参与情况及会议价值
