# 周报自动化工具 - 用户使用手册

> 版本：v1.1.0
> 更新日期：2026-03-22

---

## 一、功能简介

本工具自动完成团队周报的汇总和提交流程：

```
读取周报 → 整合数据 → 大模型优化 → 自动提交
```

支持 Windows、Linux、macOS 三个平台。

---

## 二、环境要求

使用前请确认以下条件已满足：

| 依赖项 | 是否必须 | 说明 |
|--------|---------|------|
| **Chrome 或 Edge** | 必须 | 需要 Chromium 内核浏览器（Safari/Firefox 不支持）。Windows 一般已有 Edge，macOS 需手动安装：`brew install --cask google-chrome` 或从 google.com/chrome 下载 |
| **内网 / VPN** | 必须 | 需连接百度内网才能访问 OKR 系统。如页面显示"无法访问此网站"，请检查网络或 VPN |
| **OKR 系统登录** | 首次需要 | 程序首次运行会打开 UUAP 登录页，需在 5 分钟内完成扫码/账号密码登录。登录状态保存在 `chrome_data/` 目录，后续自动复用 |
| **大模型平台登录** | 视情况 | web 模式：首次需在浏览器中登录大模型平台（如百度 OneAPI）。API 模式：需在配置中填写 API Key，无需浏览器登录 |
| **Python 3.8+** | 仅 macOS 源码版 | Windows/Linux 版已打包为独立程序，无需 Python。macOS 源码版需安装：`brew install python@3.12` |

## 三、首次使用

### 1. 解压

将收到的 `WeeklyReportTool_v1.1.0_{平台}.zip` 解压到任意目录。

### 2. 启动程序

| 平台 | 启动方式 |
|------|---------|
| Windows | 双击 `WeeklyReportTool.exe` |
| Linux | 终端运行 `./WeeklyReportTool` |
| macOS | 终端运行 `./WeeklyReportTool` |

> Linux/macOS 首次需授予执行权限：`chmod +x WeeklyReportTool`
> macOS 如遇安全拦截：`xattr -cr WeeklyReportTool_v*/` 或在「系统设置 > 隐私与安全」中允许

程序启动后会自动：
1. 打开 Chrome 浏览器
2. 在浏览器中显示配置页面

### 3. 配置

在配置页面中填写以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **员工ID** | 你的工号 | `s673090` |
| **团队成员ID** | 需要汇总周报的所有成员工号 | `s673090, s801573, ...` |
| **目标周** | 0=当周，1=上周，2=上上周 | `0` |
| **大模型平台** | 用于优化周报的 AI 平台 | `BO`（百度 OneAPI） |

填写完成后点击「保存配置」。

### 4. 启动执行

勾选「已阅读并同意用户协议」，点击「启动」按钮。

## 四、执行流程

程序自动按 4 个步骤执行，进度页面实时显示：

| 步骤 | 名称 | 说明 | 耗时参考 |
|------|------|------|---------|
| 1 | **读取周报** | 登录 OKR 系统，逐个抓取团队成员周报 | 2-5 分钟 |
| 2 | **整合周报** | 合并所有成员数据，生成结构化 JSON | 几秒 |
| 3 | **优化周报** | 调用大模型优化周报内容和格式 | 1-3 分钟 |
| 4 | **提交周报** | 自动填写并提交优化后的周报 | 1-2 分钟 |

### 登录说明

**首次使用**时，步骤 1（OKR 系统）和步骤 3（大模型平台）可能需要手动登录：

1. 程序会打开浏览器并跳转到登录页面
2. 请在浏览器中完成登录（扫码/账号密码）
3. 登录成功后程序自动继续
4. 登录状态会保存在 `chrome_data/` 目录，下次无需重复登录

> 如果 5 分钟内未完成登录，程序会超时退出。

### 浏览器断开处理

如果运行过程中浏览器意外关闭或断开连接，程序会自动尝试重连（最多 3 次）。重连失败则退出程序，重新启动即可。

## 五、配置详解

配置文件位于 `config/config.json`，可通过配置页面修改，也可直接编辑。

### 基本配置

```json
{
  "user": {
    "my_id": "s673090",
    "staff_ids": ["s673090", "s801573", "s733504"]
  },
  "weekly_report": {
    "target_week": 0,
    "default_platform": "BO"
  }
}
```

### 大模型平台

| 代号 | 平台 | 调用方式 | 说明 |
|------|------|---------|------|
| `BO` | 百度 OneAPI | web/api | 默认平台 |
| `DS` | DeepSeek | web/api | 需要 API Key |
| `DB` | 豆包 | web/api | 需要 API Key |
| `KM` | Kimi | web/api | 需要 API Key |

- **web 模式**：通过浏览器自动化操作网页版（首次需登录）
- **api 模式**：直接调用 API 接口（需配置 `api_key`）

### API 配置示例

在 `config.json` 的 `llm_platforms` 中配置：

```json
"DS": {
  "name": "DeepSeek",
  "type": "api",
  "url": "https://chat.deepseek.com",
  "api_config": {
    "base_url": "https://api.deepseek.com",
    "api_key": "sk-your-api-key-here",
    "model": "deepseek-chat",
    "timeout": 300,
    "max_tokens": 16000,
    "temperature": 0.7
  }
}
```

### 高级配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `browser.mode` | `headful` | 浏览器模式：`headful`=可见，`headless`=无头 |
| `browser.screenshot_on_error` | `true` | 出错时自动截图到 `scripts/browser_debug/` |
| `logging.debug` | `false` | 开启后显示详细调试信息 |
| `logging.save_intermediate_files` | `true` | 保存中间文件（原始/优化后的周报 JSON） |

## 七、常见问题

### Q1: 浏览器没有自动打开

- 检查是否已安装 Chrome 或 Edge（Safari/Firefox 不支持）
- macOS 未安装时程序会自动打开 Chrome 下载页
- 手动在浏览器中访问 `http://localhost:8081`

### Q2: 提示"无法访问此网站"

- 检查是否已连接百度内网或 VPN
- 检查网络连接是否正常

### Q2: 提示"登录失败"或"登录超时"

- 首次使用时需在弹出的浏览器窗口中手动完成登录
- 确保在 5 分钟内完成登录操作
- 登录完成后程序会自动继续

### Q3: 大模型优化失败

- **web 模式**：检查大模型平台是否需要重新登录
- **api 模式**：检查 `config.json` 中的 `api_key` 是否正确
- 检查网络连接是否正常

### Q4: 进度页面停滞不动

- 检查进度提示信息，可能正在等待登录
- 如果提示"浏览器连接断开"，程序会自动重连
- 重连 3 次失败后程序退出，请重新启动

### Q5: 提交失败

- 确认 OKR 系统页面正常加载
- 确认当周周报未被锁定
- 查看 `logs/` 目录下的日志获取详细错误

### Q6: 如何查看详细日志

- 设置 `config.json` 中 `logging.debug` 为 `true`
- 重新运行程序，控制台和进度页面会显示详细信息
- 日志文件位于 `logs/` 目录

### Q7: 如何修改 LLM 优化提示词

编辑 `config/prompts/WeeklyReportOptimizePrompt.md`，该文件定义了大模型优化周报时使用的提示词模板。

### Q8: 如何只抓取不提交 / 只提交不抓取

目前工具按完整流程执行（抓取 → 优化 → 提交）。如需单独执行某个步骤，可通过命令行直接运行对应脚本（需 Python 环境）。
