# 周报自动化工具 - 打包手册

> 版本: 1.1.0 | 更新日期: 2026-03-22

---

## 一、概述

本手册面向**开发者/维护者**，介绍如何在 Windows、Linux、macOS 三个平台上打包发布周报自动化工具。

**核心约束**：PyInstaller 不支持交叉编译，必须在目标平台上执行打包。当前开发环境为 WSL（Linux），可同时调用 Windows 原生 Python 完成 Windows 打包。

---

### 共享模块（Git 外部依赖）

以下模块由 `build.py` 从外部目录自动复制，不在本项目 Git 中：

| 模块 | 来源 | 说明 |
|------|------|------|
| `llmchat_final.py` | `TokenLess/scripts/` | LLM 网页自动化 |
| `llmapi_v10.py` | `TokenLess/scripts/` | LLM API 调用 |
| `browser_automation.py` | `libs/` | 浏览器管理（CDP 连接/重连） |

缺少时 `build.py` 会打印警告，打包产物缺少对应功能。

## 四、三平台并行打包（推荐）

在 WSL 环境下，可同时执行三个平台的打包，互不干扰：

```bash
cd JobsOfAndy/BaiduOKR_WeeklyReport

# ① Windows — 调用宿主机 Python（生成 .exe）
"/mnt/c/Program Files/Python312/python.exe" scripts/build.py &

# ② Linux — 使用 WSL 本地 Python
python3 scripts/build.py &

# ③ macOS — 导出源码包（需拷贝到 Mac 上编译）
python3 scripts/build.py --export-mac &

wait  # 等待全部完成
```

**平台隔离**：`build.py` 自动将工作目录、dist 目录、release 目录按平台分离（`work_windows`/`work_linux`、`dist_windows`/`dist_linux`、`..._windows_pyinstaller`/`..._linux_pyinstaller`），三个构建可安全并行。

### 输出产物

| 平台 | 发布目录 | ZIP 文件 | 说明 |
|------|---------|---------|------|
| Windows | `release/WeeklyReportTool_v1.1.0_windows_pyinstaller/` | `..._windows.zip` | 包含 `.exe`，双击即用 |
| Linux | `release/WeeklyReportTool_v1.1.0_linux_pyinstaller/` | `..._linux.zip` | ELF 可执行文件 |
| macOS | — | `WeeklyReportTool_mac_source.zip` | 源码包，需在 Mac 上打包 |

### macOS 最终打包

将 `WeeklyReportTool_mac_source.zip` 拷贝到 Mac 后：

```bash
unzip WeeklyReportTool_mac_source.zip
cd WeeklyReportTool_mac_source
chmod +x scripts/build_mac.sh
./scripts/build_mac.sh
```

详见 `docs/mac-packaging-guide.md`。

## 五、单平台打包

```bash
# 仅检查环境，不打包
python scripts/build.py --check

# PyInstaller 打包（默认）
python scripts/build.py

# 指定打包方式
python scripts/build.py --method pyinstaller
```

Windows 也可使用打包向导：

```cmd
cd scripts
打包.bat
```

## 六、打包流程详解

`build.py` 自动执行以下步骤：

| 步骤 | 说明 |
|------|------|
| 1. 检查 Python 版本 | 需要 3.8+ |
| 2. 检查项目目录 | `scripts/`、`config/` 必须存在 |
| 3. 检查打包工具 | PyInstaller 是否安装 |
| 4. 安装依赖 | 自动安装缺失的打包工具 |
| 5. 准备源码 | 复制到 `build/work_{platform}/`，含共享模块 |
| 6. 代码保护 | PyArmor 混淆（小文件）+ .pyc 编译（大文件 fallback） |
| 7. PyInstaller 打包 | onedir 目录模式，输出到 `build/dist_{platform}/` |
| 8. 创建发布包 | 复制 exe + config + scripts 到 `release/..._pyinstaller/` |
| 9. 创建 ZIP | 压缩为 `WeeklyReportTool_v{VER}_{platform}.zip` |

## 七、发布包结构

```
WeeklyReportTool_v1.1.0_windows_pyinstaller/
├── WeeklyReportTool.exe        # 主程序（Linux/Mac 无后缀）
├── _internal/                  # PyInstaller 运行时
│   ├── config/                 # 配置副本（只读，勿改）
│   ├── scripts/                # 脚本副本
│   └── playwright/             # Playwright 驱动
├── config/                     # ⭐ 用户配置（可编辑）
│   ├── config.json
│   ├── prompts/
│   └── browser_state.json      # 运行时生成
├── scripts/                    # 受保护脚本（.pyc / .py）
│   └── browser_debug/          # 错误截图（运行时生成）
├── chrome_data/                # 浏览器登录数据（运行时生成）
└── logs/                       # 运行日志
```

**路径优先级**：程序优先读取 exe 同级的 `config/`（用户可编辑），而非 `_internal/` 中的打包副本。更新程序时保留 `config/` 和 `chrome_data/` 即可保持配置和登录状态。

## 九、代码保护

| 方式 | 保护强度 | 说明 |
|------|---------|------|
| PyArmor 混淆 | 中 | 小于 32KB 的文件自动使用 |
| .pyc 字节码编译 | 低 | 大文件的 fallback |

PyArmor 试用版对 >32KB 文件有限制，`build.py` 自动回退到 `.pyc`。此警告可安全忽略。

## 十、各平台注意事项

### Windows

- 输出 `WeeklyReportTool.exe`（GUI 模式，无控制台窗口）
- 用户可能需要安装 Visual C++ Redistributable

### Linux

- 输出 ELF 可执行文件，运行前需：`chmod +x WeeklyReportTool`
- 需要安装 Chrome/Chromium：`sudo apt install google-chrome-stable`

### macOS

- 需要 Xcode Command Line Tools：`xcode-select --install`
- Gatekeeper 拦截时：`xattr -cr release/WeeklyReportTool_v*/`
- Apple Silicon 和 Intel 构建不通用，需在对应架构上打包

## 十一、常见问题

### 打包失败

1. 确认 Python 版本：`python --version`（需 3.8+）
2. 确认依赖：`pip install pyinstaller pyarmor`
3. 查看 `build.py` 输出的详细错误

### 并行打包冲突

不会发生。`build.py` 使用平台隔离的目录（`work_windows`/`work_linux`、`dist_windows`/`dist_linux`），可安全并行。

### 打包后运行报错

1. 终端直接运行可执行文件查看报错
2. 检查 `config/config.json` 配置
3. 查看 `logs/` 目录下的日志

### 更新发布

1. 修改源码
2. 升级版本号：编辑 `scripts/version.py`
3. 重新执行三平台打包
4. 分发 `release/*.zip`
5. 用户替换整个目录，保留 `config/` 和 `chrome_data/`

### 清理构建产物

```bash
rm -rf build/ dist/    # 中间产物
rm -rf release/        # 发布包（分发后可删）
```
